# Vocaly

Vocaly is a playful singing practice app built around bite-sized lessons, real-time pitch feedback, and AI vocal coaching. Learners sign in, follow a guided lesson journey, record lesson attempts, earn stars, and can also use free practice mode to sing anything and receive structured feedback.

## Features

- Guided singing journey with locked lesson progression
- Beginner lesson catalog covering breath, pitch matching, scales, intervals, and melody
- Web Audio target tone playback for each lesson
- Real-time microphone pitch detection and pitch-track visualization
- Attempt scoring based on target-note hit rate
- AI vocal coaching through the Lovable AI Gateway
- Free practice mode for open-ended singing feedback
- Supabase authentication, profiles, lesson attempts, progress, and streak tracking
- Google OAuth through Lovable cloud auth

## Getting Started

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Preview the production build:

```bash
npm run preview
```

## Environment Variables

Create a local `.env` file with the Supabase and AI Gateway values used by the app.

```bash
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_PUBLISHABLE_KEY=your_supabase_publishable_key

SUPABASE_URL=your_supabase_project_url
SUPABASE_PUBLISHABLE_KEY=your_supabase_publishable_key

LOVABLE_API_KEY=your_lovable_ai_gateway_key
```

The browser client reads the `VITE_` variables. Server-rendered code and server functions can also use the non-prefixed Supabase variables. AI voice analysis requires `LOVABLE_API_KEY`.

## Supabase

The project includes Supabase migrations in `supabase/migrations`.

The schema creates:

- `profiles`: display name, practice streak, and last practice date
- `lesson_attempts`: pitch score, AI score, overall score, and AI feedback JSON
- `lesson_progress`: best score, completion state, and earned stars per lesson
- Row-level security policies so users can only access their own data
- A signup trigger that creates a profile for each new auth user

Apply migrations with the Supabase CLI from the project root:

```bash
supabase db push
```

The Supabase project id is configured in `supabase/config.toml`.

## App Routes

- `/`: landing page
- `/auth`: email/password auth and Google OAuth
- `/journey`: authenticated lesson map, progress, stars, and streak
- `/lesson/$lessonId`: guided lesson player, pitch capture, scoring, and AI feedback
- `/practice`: free practice recorder with open-ended AI coaching

## Deployment

This app is configured for Cloudflare Workers through `wrangler.jsonc`.

```bash
npm run build
npx wrangler deploy
```

Before deploying, configure the required runtime secrets for Supabase and `LOVABLE_API_KEY` in the target Cloudflare environment.

## Notes

- Microphone access is required for lesson recording, pitch detection, and free practice.
- AI feedback is generated by `google/gemini-2.5-flash` through the Lovable AI Gateway.
- The lesson catalog currently lives in `src/lib/lessons.ts`.
- Pitch detection is implemented client-side with an autocorrelation-based detector in `src/lib/pitch.ts`.
